# 增长与商业化计划

更新时间：2026-07-01

本计划用于承接 `AI 前沿情报站 / Frontier Intel` 后续产品化、渠道推广、用户增长、转化和广告接入工作。工程实现仍以 `docs/fullstack-implementation-plan.md` 和 `docs/implementation-roadmap.md` 为准；本文件只定义产品增长和商业化方向。

## 核心判断

项目有竞争性，但不能走普通 AI 工具导航站路线。

市面上同类产品大致分为四类：

- AI 工具目录：强在收录量、分类和搜索，例如大型 AI 工具导航站。
- 模型榜单：强在模型评分、价格、速度、上下文、真实用量或 benchmark。
- AI 新闻媒体：强在内容更新频率、社群传播和观点包装。
- 社区热榜：强在早期发现和讨论热度。

本项目的差异点应放在“可验证的 AI 前沿情报系统”：

- 不拼工具数量，拼信号质量。
- 不只展示榜单，展示榜单背后的来源和评分解释。
- 不只转述新闻，区分官方事实、文档变更、平台数据、榜单评测、社区热度和传闻。
- 不隐藏数据源失败，把来源健康、缺失密钥、更新时间和证据链作为核心功能。
- 面向中文用户提供“今天发生了什么、可信度多高、影响谁、我该不该行动”的判断。

## 目标用户

第一批用户应聚焦在高频关注 AI 变化的人群：

- AI 产品经理：需要追踪模型、Agent、工具变化，辅助产品选型。
- 开发者和独立开发者：需要判断模型、AI 编程工具、MCP / Skill 插件是否值得试。
- AI 自媒体和培训从业者：需要稳定素材、证据链和周报结构。
- 企业技术负责人：需要判断某个模型、Agent 或工具是否适合进入内部试点。
- 投资、咨询和行业研究人员：需要早期信号、趋势变化和可复查来源。

不建议早期面向泛大众做“AI 工具大全”。泛用户留存弱，内容成本高，也容易陷入同质化竞争。

## 产品路线

### 第一阶段：Web / H5 公开版

目标：先把公开可访问的 Web/H5 跑通，作为内容传播、搜索收录和小程序承接前的主阵地。

核心页面：

- 今日 AI 前沿：每天 3 到 5 条高价值信号。
- 信号详情：结论、可信度、影响、证据链、来源类型。
- 模型榜单：模型能力、价格、上下文、速度和可用性变化。
- Agent 榜单：浏览器 Agent、代码 Agent、电脑使用 Agent、研究 Agent。
- Skill / 插件热度：MCP Server、Agent Skill、连接器、开发工具。
- 可信来源：来源健康、更新时间、失败原因和待配置状态。

验收标准：

- 每条重点信号都有可点击证据 URL。
- 首页能在 30 秒内让用户知道今天最重要的 AI 变化。
- 数据刷新失败时页面明确显示降级或过期状态。
- 移动端能顺畅阅读和分享。

### 第二阶段：微信小程序

目标：承接公众号和私域流量，提供轻量、高频、可留存的使用场景。

首版功能：

- 今日：每日重点信号。
- 榜单：模型、Agent、工具、插件。
- 详情：证据链、可信度、来源类型。
- 收藏：保存关注的模型、工具或信号。
- 订阅：按关键词、厂商、类别订阅提醒。
- 周报：生成一周 AI 情报摘要。

小程序不建议一开始复制完整工作台。它应该是高频入口，重点服务“快速看、快速存、快速订阅”。

### 第三阶段：公众号内容矩阵

目标：用稳定栏目建立用户认知，把公众号变成解释和转化入口。

固定栏目：

- 每日短报：`AI 前沿 5 分钟`，每天 3 条信号，每条包含结论、可信度、证据、影响。
- 每周榜单：模型、Agent、AI 编程工具和 MCP / Skill 生态变化。
- 选型文章：例如某个模型是否值得切换、Codex 和 Claude Code 的适用边界。
- 避坑文章：拆解传闻、营销榜单和不可靠来源。
- 案例文章：企业、开发者或自媒体如何使用情报站做决策。

每篇文章的转化动作：

- 查看完整证据链。
- 进入小程序收藏。
- 订阅关键词提醒。
- 下载或生成周报。
- 加入社群或邮件列表。

### 第四阶段：抖音内容入口

目标：做发现和破圈，不把抖音当完整产品承载平台。

内容形式：

- 30 秒看懂今天 AI 圈最大变化。
- 这个新模型到底是不是营销噱头。
- 程序员本周最值得试的 3 个 AI 工具。
- 不要只看跑分，模型选择看这几个指标。
- 一分钟解释某个 Agent / MCP / AI 编程工具的真实用途。

转化动作：

- 评论关键词获取榜单或报告。
- 引导进入小程序查看证据链。
- 引导关注公众号看深度分析。
- 引导加入社群或订阅每周报告。

抖音内容要短、具体、有判断，不做泛泛的新闻转述。

### 第五阶段：App

App 不作为早期优先项。只有满足以下条件后再考虑：

- 小程序或 Web 已有稳定日活。
- 用户有强推送、收藏、长期订阅和多设备同步需求。
- 已经验证付费订阅或团队版有人买单。
- 有足够资源维护 iOS、Android、审核、推送和广告 SDK。

早期更推荐使用 Web/H5 + 微信小程序 + 公众号 + 抖音的组合。

## 增长策略

### 内容获客

内容不以“AI 新闻”作为主标题，而以“判断和行动建议”作为主标题。

推荐标题结构：

- `今天 AI 圈最值得关注的 3 个变化`
- `这个模型更新对开发者真正有什么影响`
- `某工具突然爆火，但证据链只有这几类来源`
- `本周 AI 编程工具榜单变化：谁更值得试`
- `企业选型：什么时候该从旧模型切到新模型`

每篇内容都要保留同一套结构：

1. 结论。
2. 可信度。
3. 证据。
4. 影响。
5. 建议行动。
6. 进入产品查看完整数据。

### 产品内增长

需要把产品设计成可分享、可收藏、可订阅：

- 每条信号生成分享卡片。
- 每个模型、Agent、工具有独立详情页。
- 每个榜单支持生成周榜图片或短链接。
- 用户可订阅厂商、类别、关键词和可信度阈值。
- 每周自动生成个人关注报告。

### 社群增长

社群不应只做闲聊群，而要做“情报反馈群”：

- 用户提交新信号。
- 用户反馈来源错误。
- 用户投票是否关注某类数据源。
- 每周收集选型问题，反向生成内容和产品功能。

## 转化路径

### 免费层

目标：降低使用门槛，建立可信品牌。

包含：

- 今日重点信号。
- 基础榜单。
- 基础来源证据。
- 公开发布日历。
- 部分历史数据。

### 注册层

目标：建立留存和用户画像。

包含：

- 收藏信号。
- 订阅关键词。
- 每周报告。
- 浏览历史。
- 个性化关注类别。

### 付费个人层

目标：验证用户是否愿意为节省时间和判断质量付费。

可提供：

- 无广告。
- 更完整历史趋势。
- 自定义监控关键词。
- 模型和工具对比。
- 高可信度提醒。
- 周报导出。
- 专题报告。

### 团队版 / B 端层

目标：提高客单价。

可提供：

- 团队关注列表。
- 企业内部选型报告。
- API 或 webhook 推送。
- 自定义来源。
- 周报自动发送到企业微信、飞书或邮箱。
- 按行业定制 AI 工具监控。

### 内容服务层

目标：利用情报系统反哺内容生产。

可提供：

- 给 AI 自媒体的选题库。
- 给培训机构的每周素材包。
- 给咨询和研究用户的趋势报告。
- 给企业客户的月度 AI 变化简报。

## 广告接入策略

广告不建议作为早期核心商业模式。原因：

- 早期流量小，广告收入有限。
- 广告会影响可信情报产品的专业感。
- 若广告和榜单边界不清，会直接损害信任。

推荐顺序：

1. 先验证内容留存和订阅。
2. 再验证个人付费或团队报告。
3. 流量稳定后接入广告。
4. 广告必须和榜单、评分、推荐结果明确隔离。

### 微信小程序广告

可考虑官方流量主能力和广告组件。

接入原则：

- 广告位只放在列表自然断点、文章底部、报告结束页。
- 不在信号可信度、榜单排名和证据链中混入广告。
- 推广内容必须标注广告或赞助。
- 激励广告只适合解锁非核心权益，例如额外导出次数，不应解锁事实信息。

实施前必须复核微信公众平台、微信小程序流量主后台和广告组件文档的最新开通条件、广告形态、结算规则和内容规范。

### 抖音小程序广告

可考虑抖音开放平台商业化变现能力。

接入原则：

- 抖音小程序更适合承接短视频用户的轻量查询和报告领取。
- 广告位放在结果页、榜单底部或工具页，不影响核心阅读。
- 视频内容和小程序落地页必须一致，避免标题党导致低留存。

实施前必须复核抖音开放平台、巨量引擎和相关广告组件文档的最新要求。

### App 广告

如果后续做 App，可考虑穿山甲、腾讯广告联盟等广告 SDK。

接入原则：

- iOS 和 Android 分别评估 SDK、隐私合规和审核风险。
- 数字订阅、会员权益和付费内容需要按应用商店规则处理。
- 广告 SDK 接入前必须补隐私政策、权限说明、数据采集说明和关闭个性化广告入口。

## 关键指标

### 内容指标

- 公众号阅读完成率。
- 公众号收藏率。
- 公众号文章到小程序点击率。
- 抖音完播率。
- 抖音评论关键词转化率。
- 内容带来的注册或订阅数。

### 产品指标

- 日活用户。
- 周活用户。
- 首次访问到第二天回访率。
- 收藏率。
- 订阅关键词设置率。
- 信号详情打开率。
- 证据 URL 点击率。
- 周报生成率。

### 商业指标

- 注册转化率。
- 免费用户到付费用户转化率。
- 付费留存。
- 团队版咨询数。
- 广告展示量、点击率和 eCPM。
- 广告对留存和信任的负面影响。

## 90 天执行计划

### 0 到 30 天：可信数据和公开入口

目标：先让产品可信、可访问、可分享。

任务：

- 修复高优先级数据源错误，降低 `sourceRuns` error 比例。
- 每条重点信号补具体证据 URL。
- 建立今日 AI 前沿公开页。
- 建立信号详情页分享卡片。
- 建立公众号每日短报模板。
- 建立 30 天内容选题表。

验收：

- 每天可以稳定产出 3 到 5 条重点信号。
- 公众号文章能稳定引导到 Web/H5 或小程序预约页。
- 用户能从分享卡片打开对应信号详情。

### 31 到 60 天：小程序 MVP 和订阅

目标：建立微信生态留存。

任务：

- 开发微信小程序 MVP。
- 接入收藏、关键词订阅、基础周报。
- 公众号文章底部固定小程序入口。
- 抖音短视频开始固定栏目。
- 建立用户反馈和信号提交入口。

验收：

- 小程序能完成今日、榜单、详情、收藏、订阅闭环。
- 公众号到小程序转化可统计。
- 用户能订阅至少一种关键词或类别。

### 61 到 90 天：付费验证和广告预埋

目标：验证商业模式，不急于大规模广告。

任务：

- 上线个人付费权益试验。
- 上线周报导出或专题报告。
- 访谈第一批高频用户。
- 设计团队版报价和试点服务。
- 评估微信小程序广告位和规则。
- 预留广告位，但默认不开启或只灰度开启。

验收：

- 至少验证一种用户愿意付费的权益。
- 明确广告是否影响留存和信任。
- 形成下一阶段商业化决策。

## 风险和边界

- 不把传闻包装成事实。低可信信号必须降权展示。
- 不为了增长牺牲证据链。流量内容也必须能回到来源。
- 不让广告影响榜单排名。广告、赞助和推荐必须分层展示。
- 不早期投入重 App。App 应建立在已验证留存和付费之后。
- 不为少量运营需求引入复杂后端。先用现有 JSON、MySQL 和定时任务跑通。
- 不在文档中写死平台广告开通条件。微信、抖音和应用商店规则变化频繁，实施前必须现场复核。

## 与工程计划的关系

本计划依赖以下工程能力：

- 稳定的数据刷新和 `sourceRuns` 状态。
- 完整证据链和来源类型分层。
- 可分享的信号详情页。
- 可移动端使用的 H5 页面。
- 小程序可复用的数据接口或静态 JSON 输出。
- 用户系统、收藏、订阅和周报生成。
- 广告位与核心榜单、证据链、评分逻辑隔离。

对应工程阶段应补充到 `docs/implementation-roadmap.md`，并在实现前拆成具体 issue 或任务单。
